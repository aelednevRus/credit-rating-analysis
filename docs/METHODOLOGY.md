# Методология расчёта кредитного рейтинга

Документ описывает методику, реализованную в модуле `credit_analysis/`
(`metrics.py`, `extract.py`, `conclusions.py`). Методика построена на данных
бухгалтерской отчётности по РСБУ (баланс + отчёт о финансовых результатах)
и относится к классу **взвешенных балльных (рейтинговых) моделей** —
аналог подходов, используемых банками при экспресс-оценке заёмщика.

## 1. Источники данных

Входные данные — выгрузка БФО в формате `.xlsx` с тремя листами:
«Сведения об организации», «Бухгалтерский баланс», «Отчет о финансовых
результатах». Для расчёта используются 3 отчётные даты баланса (текущий,
предыдущий, позапрошлый год — нужен для средних величин в ROE/ROA) и
2 периода ОФР (текущий, предыдущий).

Используемые строки отчётности (коды по РСБУ):

| Показатель | Код строки | Откуда |
|---|---|---|
| Оборотные активы | 1200 | Баланс |
| Запасы | 1210 | Баланс |
| НДС по приобретённым ценностям | 1220 | Баланс |
| Денежные средства | 1250 | Баланс |
| Краткосрочные финансовые вложения | 1240 | Баланс |
| Краткосрочные обязательства | 1500 | Баланс |
| Капитал и резервы (СК) | 1300 | Баланс |
| Активы, всего | 1600 | Баланс |
| Долгосрочные обязательства | 1400 | Баланс |
| Долгосрочные заёмные средства | 1410 | Баланс |
| Краткосрочные заёмные средства | 1510 | Баланс |
| Выручка | 2110 | ОФР |
| Прибыль от продаж | 2200 | ОФР |
| Проценты к уплате | 2330 | ОФР |
| Чистая прибыль | 2400 | ОФР |
| Финансовые вложения (внеоборотные) | 1170 | Баланс |
| Внеоборотные активы | 1100 | Баланс |
| Прочие доходы | 2340 | ОФР |

Последние три строки не участвуют в расчёте 11 коэффициентов напрямую —
они используются модулем `conclusions.py` для качественных выводов
(признак холдинговой модели, качество прибыли — см. раздел 6).

Отрицательные суммы в отчётности (в круглых скобках) и пропуски (`-`)
нормализуются в числа автоматически (`extract._to_number`).

## 2. Финансовые коэффициенты

Рассчитываются 11 коэффициентов + чистая прибыль как отдельный индикатор,
за текущий и предыдущий год (для оценки динамики).

| Показатель | Формула | Норматив |
|---|---|---|
| Текущая ликвидность | Оборотные активы / Краткосрочные обязательства | ≥ 1,5 |
| Быстрая ликвидность | (Обор. активы − Запасы − НДС) / Кратк. обязательства | ≥ 1,0 |
| Абсолютная ликвидность | (Ден. средства + Кратк. фин. вложения) / Кратк. обязательства | ≥ 0,2 |
| Автономия | Собственный капитал / Активы | ≥ 0,4 |
| Финансовый рычаг | (Долгоср. + Кратк. заёмные средства) / СК | ≤ 1,0 |
| Долг / Активы | Обязательства всего / Активы | ≤ 0,7 |
| Покрытие процентов (ICR) | Прибыль от продаж / Проценты к уплате\*\* | ≥ 1,5 |
| Оборачиваемость активов | Выручка / Активы | чем выше, тем лучше |
| ROE | Чистая прибыль / Среднегод. СК* | ≥ 15% |
| ROA | Чистая прибыль / Среднегод. активы* | чем выше, тем лучше |
| Рентабельность продаж | Прибыль от продаж / Выручка | чем выше, тем лучше |
| Чистая прибыль, тыс. руб. | — | > 0 |

\* Средняя величина = (значение на конец текущего периода + значение на
конец предыдущего периода) / 2.

\*\* При нулевых процентах к уплате (компания не платит проценты по долгу)
ICR принимает sentinel-значение **999** — трактуется как «нет долга», а не
как деление на ноль (`metrics.compute_ratios`).

## 3. Балльная оценка (светофор)

Каждый показатель получает **0 / 0,5 / 1 балл** по двум пороговым
значениям (`THRESHOLDS` в `metrics.py`): полный порог → 1 балл,
половинный порог → 0,5 балла, ниже — 0 баллов. Для показателей «чем
меньше, тем лучше» (финансовый рычаг, Долг/Активы) логика зеркальная.

| Ключ | Полный балл (1,0) | Половина (0,5) |
|---|---|---|
| current_ratio | ≥ 1,5 | ≥ 1,0 |
| quick_ratio | ≥ 1,0 | ≥ 0,7 |
| absolute_ratio | ≥ 0,2 | ≥ 0,1 |
| autonomy | ≥ 0,4 | ≥ 0,2 |
| leverage | ≤ 1,0 | ≤ 2,0 |
| debt_to_assets | ≤ 0,7 | ≤ 0,85 |
| icr | ≥ 1,5 | ≥ 0,75 |
| turnover | ≥ 0,1 | ≥ 0,05 |
| roe | ≥ 15% | ≥ 7,5% |
| roa | ≥ 2% | ≥ 0,5% |
| margin | ≥ 20% | ≥ 10% |
| net_profit | > 0 | = 0 |

Балл каждого показателя визуализируется цветом «светофора»:
зелёный (1,0) / жёлтый (0,5) / красный (0,0).

## 4. Комплексный показатель кредитоспособности

Итоговый балл — **взвешенная сумма 6 ключевых факторов** (не всех 12
коэффициентов — остальные носят справочный/поясняющий характер):

| Фактор | Вес |
|---|---|
| Покрытие процентов (ICR) | 25% |
| Текущая ликвидность | 20% |
| Автономия | 20% |
| Долг / Активы | 15% |
| ROE | 15% |
| Оборачиваемость активов | 5% |

```
Комплексный балл = 100 × Σ (вес_i × балл_i),   балл_i ∈ {0; 0,5; 1}
```

Итог приводится к шкале **0–100** и рассчитывается отдельно для текущего
и предыдущего года — это даёт динамику класса кредитоспособности.

## 5. Классы кредитоспособности

| Балл | Класс |
|---|---|
| ≥ 70 | A — высокая кредитоспособность |
| 50–69 | B — приемлемая |
| 30–49 | C — пониженная |
| < 30 | D — низкая (высокий риск) |

## 6. Качественные выводы

Помимо числовой оценки, модуль `conclusions.py` формирует текстовые
выводы: динамика класса, состояние ликвидности, долговой нагрузки,
покрытия процентов и качества прибыли. Отдельно проверяется контекстный
признак **холдинговой модели** — если долгосрочные финансовые вложения
(код 1170) превышают 50% активов (код 1600), в отчёт добавляется оговорка о
необходимости анализа консолидированной отчётности группы, так как
экстремальные значения коэффициентов у головной компании холдинга не
всегда отражают операционные риски.

Признак сознательно ограничен одним критерием: другие потенциальные
индикаторы холдинговой структуры (например, вознаграждение за выданные
поручительства) не публикуются в стандартной 3-листовой выгрузке БФО
(баланс + ОФР) и не могут быть надёжно извлечены без ручного анализа
пояснений к отчётности.

Также проверяется «качество прибыли»: если разовые доходы (например,
списание кредиторской задолженности) формируют более 20% чистой
прибыли текущего периода, это отмечается как фактор, снижающий
устойчивость результата.

## 7. Ограничения методики

- Методика рассчитана на нефинансовые организации, отчитывающиеся по
  РСБУ (баланс + ОФР); не подходит для банков, страховых компаний и
  МСФО-отчётности без адаптации.
- Пороговые значения — общеотраслевые ориентиры, не учитывают отраслевую
  специфику (например, у торговых компаний нормальная текущая
  ликвидность ниже, чем у производственных).
- Веса 6 факторов заданы экспертно, а не откалиброваны статистически
  (регрессией на выборке дефолтов) — методика носит балльный, а не
  probability-of-default характер.
- Не учитывает качественные факторы вне отчётности: кредитную историю,
  деловую репутацию, судебные споры, обеспечение по кредиту, отраслевые
  и макроэкономические риски.
- Для холдинговых структур требует ручной проверки консолидированной
  отчётности (см. п. 6).

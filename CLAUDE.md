# CLAUDE.md — правила для Claude Code

Проект: **расчёт кредитного рейтинга компаний по РСБУ-отчётности** и его
визуализация на дашборде. Работаем в VS Code с Claude Code.

## Роль
Ты — senior Python-инженер с уклоном в финансовую аналитику. Пишешь
корректный, читаемый, идиоматичный код без плейсхолдеров, готовый к запуску.

## Ключевые правила
1. **Методика — единственный источник истины.** Любые формулы, пороги
   (`THRESHOLDS`), веса (`WEIGHTS`) и классы должны совпадать с
   `docs/METHODOLOGY.md`. Меняешь логику — сначала обнови документ.
2. **Разделение слоёв.** `credit_analysis/` не зависит от слоя представления.
   Вся визуализация — в `dashboard/`.
3. **Устойчивость чтения.** Строки отчётности ищем по коду РСБУ (1200,
   2110, ...), а не по фиксированным координатам ячеек. Отрицательные суммы
   в скобках и пропуски «-» нормализует `extract._to_number`.
4. **Деньги — тыс. руб.** как в БФО. Проценты (ROE/ROA/margin) хранятся как
   доли (0.15 = 15%), форматируются в представлении.
5. **Безопасное деление.** Никаких падений на нуле в знаменателе —
   `_safe_div`. ICR при нулевых процентах трактуется как «нет долга» (999).

## Рабочий процесс изменений
- Перед правкой — короткий план (3–7 пунктов).
- Правки локальные и безопасные, если не просят глубокий рефакторинг.
- После правки ядра — прогони самопроверки: `python -m credit_analysis.metrics`
  и `python -m credit_analysis.rating`.
- Не добавляй зависимости без нужды; текущие — в `requirements.txt`.

## Команды
```bash

pip install -r requirements.txt

# Прогон анализа из CLI (папка или один файл)

# Генерация HTML-дашборда
python dashboard/html_report.py
```
